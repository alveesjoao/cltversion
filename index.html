    Controle de versão

    Esta tecnologia permite que vários uploads de diferentes versões de um mesmo código seja mapeado, permitindo assim o trabalho em conjunto de vários desenvolvedores em um mesmo código, por exemplo.

    Imagine que você junto de outros quatro devs devem refatorar o mesmo código, como raios vocês fariam isso? Copiando o código editado pelo parceiro1, editando e enviando ao próximo? Isto levaria eternidades!! Já com o controle de versão todos vocês poderiam refatorar o código ao mesmo tempo, e a tecnologia (Git por exemplo) preservaria o conteúdo que não foi mudado e adequaria o código às novas versões. Portanto, com o controle de versão podemos utilizar deste benefício de trabalho em conjunto, que é um dos inúmeros da lista.

    Controle de Mudanças

    Ainda utilizando o exemplo dos colegas para refatorar um único código, vamos imaginar o seguinte: Vocês por alguma razão editaram o mesmo trecho de código! Em tese, se estivessem trabalhando a la idade das pedras isto viraria uma confusão total, certo? Com o recurso de controle de mudanças é possível identificar o trecho alterado, quem o alterou e até mesmo compara-lo com o código original, coisa que nunca seria possível se apenas copiássemos o atual codigo e repassássemos para o nosso colega.

    Integração Contínua

    Por útlimo, concluímos com a integração contínua!

    Com a tecnologia git podemos aderir à uma prática de entrega constante, a qual consite nos famosos ''commits''. Mas afinal, o que é tudo isso?

    A prática de entrega constante pode ser chamada de integração contínua, onde várias "versões" e "atualizações" são enviadas à um repositório único, o qual pode ser interpretado como uma grande pasta onde estará alocado o seu projeto.
